high Missing Origin Validation
  • Module: webpack-dev-server
  • Published: November 7th 2018
  • Reported by: Jiantao Li
  • CWE-346
  • CVE-2018-14732
  • Vulnerable: <2.11.4 || >=3.0.0 <=3.1.10
  • Patched: >=2.11.4 <3.0.0 || >=3.1.11
  • Exploitability: 5

Overview

Versions of webpack-dev-server before 3.1.10 are missing origin validation on the websocket server. This vulnerability allows a remote attacker to steal a developer's source code because the origin of requests to the websocket server that is used for Hot Module Replacement (HMR) are not validated.

Findings

  • react-scripts>webpack-dev-server

Remediation

For webpack-dev-server 2.x update to version 2.11.4 or later. For webpack-dev-server 3.x update to version 3.1.11 or later.

high Arbitrary File Overwrite
  • Module: tar
  • Published: April 4th 2019
  • Reported by: Max Justicz
  • CWE-59
  • Vulnerable: <2.2.2 || >=3.0.0 <4.4.2
  • Patched: >=2.2.2 <3.0.0 || >=4.4.2
  • Exploitability: 5

Overview

Versions of tar prior to 4.4.2 for 4.x and 2.2.2 for 2.x are vulnerable to Arbitrary File Overwrite. Extracting tarballs containing a hardlink to a file that already exists in the system, and a file that matches the hardlink will overwrite the system's file with the contents of the extracted file.

  • react-scripts>jest>jest-cli>jest-runtime>jest-haste-map>sane>fsevents>node-pre-gyp>tar

Remediation

For tar 4.x, upgrade to version 4.4.2 or later. For tar 2.x, upgrade to version 2.2.2 or later.

low Prototype Pollution
  • Module: minimist
  • Published: September 23rd 2019
  • Reported by: Checkmarx Research Team
  • CWE-471
  • Vulnerable: <0.2.1 || >=1.0.0 <1.2.3
  • Patched: >=0.2.1 <1.0.0 || >=1.2.3
  • Exploitability: 1

Overview

Affected versions of minimist are vulnerable to prototype pollution. Arguments are not properly sanitized, allowing an attacker to modify the prototype of Object, causing the addition or modification of an existing property that will exist on all objects.
Parsing the argument --__proto__.y=Polluted adds a y property with value Polluted to all objects. The argument --__proto__=Polluted raises and uncaught error and crashes the application.
This is exploitable if attackers have control over the arguments being passed to minimist.

moderate Regular Expression Denial of Service
  • Module: acorn
  • Published: March 2nd 2020
  • Reported by: Peter van der Zee
  • CWE-400
  • Vulnerable: >=5.5.0 <5.7.4 || >=6.0.0 <6.4.1 || >=7.0.0 <7.1.1
  • Patched: >=5.7.4 <6.0.0 || >=6.4.1 <7.0.0 || >=7.1.1
  • Exploitability: 3

Overview

Affected versions of acorn are vulnerable to Regular Expression Denial of Service. A regex in the form of /[x-\ud800]/u causes the parser to enter an infinite loop. The string is not valid UTF16 which usually results in it being sanitized before reaching the parser. If an application processes untrusted input and passes it directly to acorn, attackers may leverage the vulnerability leading to Denial of Service.

  • react-scripts>jest>jest-cli>jest-runtime>jest-config>jest-environment-jsdom>jsdom>acorn
  • react-scripts>jest>jest-cli>jest-environment-jsdom>jsdom>acorn
  • react-scripts>webpack>acorn
  • react-scripts>webpack>acorn-dynamic-import>acorn

Remediation

Upgrade to versions 5.7.4, 6.4.1, 7.1.1 or later.
